    Available module types¶

    This page contains a list of all available Kiara module types, and their details.

    Note

    The formatting here will be improved later on, for now this should be enough to get the important details of each module type.

    and¶

    Module: and

  class          kiara_modules.default.logic_gates.AndModule                    

  doc            Returns 'True' if both inputs are 'True'.                      

  config class   kiara_modules.default.logic_gates.LogicProcessingModuleConfig  

  config          ───────────────────────────────────────────────────────────   
                   constants   object   Value constants for this module.        
                   delay       number   the delay in seconds from processing    
                                        start to when the output is             
                                        returned.                               
                  ───────────────────────────────────────────────────────────   

  src            def process(self, inputs: StepInputs, outputs: StepOutputs) …  

                     time.sleep(self.config.delay)  # type: ignore              

                     outputs.y = inputs.a and inputs.b

    nand¶

    PipelineModule: nand

  class               kiara.modules.pipelines.Nand                              

  doc                 Returns 'False' if both inputs are 'True'.                

  config class        kiara.config.PipelineModuleConfig                         

  config               ──────────────────────────────────────────────────────   
                        constants   object   Value constants for this           
                                             module.                            
                        meta        object   Metadata for this workflow.        
                       ──────────────────────────────────────────────────────   

  pipeline inputs     a:                                                        
                        type: boolean                                           
                        doc: A boolean describing this input state.             
                      b:                                                        
                        type: boolean                                           
                        doc: A boolean describing this input state.             

  pipeline outputs    y:                                                        
                        type: boolean                                           
                        doc: A boolean describing the module output state.      

  processing stages   stage 0:                                                  
                        and:                                                    
                          module: and                                           
                          desc: Returns 'True' if both inputs are 'True'.       
                          inputs:                                               
                          - a ← __pipeline__.a                                  
                          - b ← __pipeline__.b                                  
                          outputs:                                              
                          - y                                                   
                      stage 1:                                                  
                        not:                                                    
                          module: not                                           
                          desc: Negates the input.                              
                          inputs:                                               
                          - a ← and.y                                           
                          outputs:                                              
                          - y → __pipeline__.y
